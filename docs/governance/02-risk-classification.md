# Risk Classification

Project: Impressive Recruitment Maps Agent Demo

Risk level: medium

Risk rationale:

The project does not use real data or production systems, but it includes AI-generated code, interactive visualisation, accessibility requirements, export functionality and public demo content that could mislead users if not clearly labelled as fictional.

High-risk areas:

No production, personal data, secrets, financial, legal, regulated or infrastructure access is allowed. The main risk areas are public demo accuracy, accessibility, fictional-data labelling, generated code quality and tool-access control.

Required approvers:

Project owner, technical reviewer, accessibility reviewer and AI governance reviewer.

Risk review frequency:

monthly
