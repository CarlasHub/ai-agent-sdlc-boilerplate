# Monitoring and Audit Plan

Monitoring owner: Project owner.

Metrics:

Console errors, accessibility findings, keyboard test results, reduced-motion behaviour, export failures, layout issues, broken filters, documentation gaps, rejected agent outputs and post-release defects.

Audit events to capture:

- tool calls
- data access
- file changes
- blocked actions
- approval requests
- failed tests
- rejected outputs
- incidents

Log location:

docs/audit/

Review cadence:

weekly
