# Agent Charter

Project: Impressive Recruitment Maps Agent Demo

Owner: Carla Goncalves

Agent purpose:

The AI agent workflow should initialise, document, govern and implement an interactive front-end recruitment maps demo using a controlled AI-Agent SDLC process.

Primary users:

Front-end developers, accessibility specialists, product teams, engineering managers, AI governance reviewers and technical stakeholders.

Expected outputs:

Governance documents, local front-end files, fictional sample data, test notes, accessibility notes, review summaries, release evidence, audit records and documentation updates.

Out of scope:

The agent must never use real client data, real candidate data, real employee data, personal data, secrets, paid APIs, backend services or production systems. It must not delete files, approve its own work or bypass governance gates. Direct publication is allowed only for the release-approved public GitHub Pages demo using fictional data and no secrets.

Success criteria:

The workflow is successful when the intake is completed, placeholders are resolved, governance documents are generated, risk and permissions are documented, human approval is recorded, implementation stays inside scope, tests are documented and release evidence is complete.

Business justification:

The agent is needed to demonstrate how AI-assisted development can be controlled through intake, governance documents, risk classification, tool-access mapping, approval gates, testing evidence and review before implementation.
