# SLSA-Inspired Artefact Integrity Mapping

This project uses SLSA-style concepts to improve trust in AI-generated artefacts.

| SLSA concept | Boilerplate practice |
|---|---|
| Source integrity | PR review, CODEOWNERS, protected branches recommended |
| Build integrity | CI workflows, npm audit, governance checks |
| Provenance | artefact record, audit logs, PR evidence |
| Dependency control | security check workflow, dependency approval requirement |
| Tamper resistance | branch protection and required checks recommended |
